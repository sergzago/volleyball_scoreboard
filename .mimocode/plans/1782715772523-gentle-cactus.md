# Plan: Block Control tab until game exists in DB

## Problem
The Control tab is accessible immediately after connecting to a game ID, even if the document doesn't exist in the DB yet. The user wants the Control tab to be available only after saving team data (which creates the document via upsert).

## Approach
Track whether the game document exists. If it doesn't exist on connect, block the Control tab until `saveTeamsBtn` creates it.

## Changes

### File: `js/mobile.js`

1. **Add flag** `gameDocumentExists = false` alongside `mobileGameConnected`

2. **In `connectToGame()`** — don't show `controlContent` immediately. Instead keep showing `controlLoading` ("Подключитесь к игре в настройках") on the Control tab. Only hide `controlLoading` and show `controlContent` once `gameDocumentExists` becomes `true`.

3. **In `subscribeToGame()` callback** — when subscription returns data (document exists), set `gameDocumentExists = true` and call `showControlContent()`.

4. **New function `showControlContent()`** — hides `controlLoading`, shows `controlContent`, sets `gameDocumentExists = true`. Idempotent (safe to call multiple times).

5. **In `saveTeamsBtn` handler** — after `update_db()` resolves (document created via upsert), call `showControlContent()`. Since `update_db` uses `scoreboard_query.update()` which returns a Promise, chain `.then(showControlContent)`.

6. **In tab click handler** — when switching to Control tab (`pageControl`), if `!gameDocumentExists`, prevent the switch and optionally show a message like "Сначала сохраните данные команд". Or simply let the tab show the loading spinner (current behavior), which is less disruptive.

## Implementation Detail

```
var gameDocumentExists = false;

function showControlContent() {
  if (gameDocumentExists) return;
  gameDocumentExists = true;
  hide(document.getElementById('controlLoading'));
  show(document.getElementById('controlContent'));
}

// connectToGame() — remove show(controlContent), keep show(controlLoading)
// subscribeToGame callback — after mobileScoreboardData = data, call showControlContent()
// saveTeamsBtn — chain .then(showControlContent) on update_db return value
```

## Files to modify
- `js/mobile.js` — all changes in this single file

2. Go to Teams tab → save → Control tab now shows game controls
3. Refresh with existing game ID → Control tab shows controls immediately
